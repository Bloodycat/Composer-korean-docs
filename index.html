Composer 한국어 메뉴얼

Composer를 설정하고 이용하는 방법에 대한 한국어 메뉴얼을 제공합니다.

Composer?

composer 는 php 를 개발하는데 있어서 패키지간의 의존성을 관리하는 의존성 관리자입니다. node.js 의 npm, ruby의 bundle과 같은 역활을 수행합니다.

php를 개발하는데 있어서 다양한 패키지들을 설치하고 의존성을 관리하며 Autoload를 사용하는데 Composer가 꼭 필요하다고 할 수 있습니다.

PHP 5.3.2+ 이상이면 Composer가 구동될 수 있습니다.

패키지의 검색

composer 를 통해서 설치할 수 있는 수많은 패키지들을 손쉽게 검색하는 방법은 packagist를 이용하는 방법입니다. 이곳에서 키워드나 패키지명, 벤더의 이름을 기반으로 검색하여 원하는 패키지를 손쉽게 찾을 수 있습니다.

한국어 문서의 번역

Composer의 한글 메뉴얼은 많은 분들의 공헌으로 진행되었습니다. 공헌자 확인하기 오타나 수정사항에 대해서는 @findstar 에게 연락하시거나 github 에서 Pull Request 를 보내주시면 검토후 반영됩니다.

기타 항목

  • Aliases-별칭 : 버전과 브랜치 네임의 별칭을 구성하는 법

  • 커스텀 인스톨러 : 패키지의 특정한 유형의 설치 방법을 수정하기

  • Satis 또는 Toran 프록시를 통해서 전용 패키지를 구성하는법 : 고유한 자신의 컴포저 저장소를 운영하는 방법

  • HTTP 기본 인증 사용법 : private 호스트에 위치한 패키지에 엑세스 하는 방법

  • 플러그인을 설정하고 사용하는 방법 : 컴포저의 기능을 확장하고 수정하는 방법

  • 스크립트 : 패키지를 설치한 전후에 호출되는 스크립트 콜백

  • 트러블 슈팅 : 문제점 해결

  • 벤더 바이너리와 vendor/bin 디렉토리 : 패키지의 커맨드라인 스크립트를 확인하는 방법

한글 메뉴얼에 대한 전체 자료는 Github 에서 확인 하실 수 있습니다.
